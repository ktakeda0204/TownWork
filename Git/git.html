branch

git branch ブランチ名

新しいブランチの作成を行う事が出来る。 何も指定しなければ、一番新しいコミットの部分から派生する。 新しいブランチ上でも同様にコミットして作業する事が出来る。 （例えばdebugの）作業を中断してmasterブランチに戻る事も出来る。

オプション

  • -v：ブランチのコミットハッシュ値&コミットメッセージを表示できる
  • -d：ブランチを消す

ブランチを作成した日時を確認したい

ブランチはオブジェクトではなく、単なるコミットへの参照なのでブランチそれ自体には日時の情報は含まれていない。

grep

git grep <キーワード名> <探したいブランチ名>

log

コミットの履歴を表示する。
  • --oneline：一行で簡潔な表示になる
  • --decorate：リファレンス名を出力する
  • --graph：コミットの分岐が可視化出来る

tag

コミットにタグを振って管理することが出来る。
  • 注釈付きのタグを振る：git tag -a <タグ名> -m “コメント、注釈”
リモートリポジトリに反映するには、普通にコミットするだけでは不十分。git push origin <タグ名>; git push origin --tagsでようやく反映される。

reflog

リポジトリとブランチのHEADの履歴を、参照ログ（reflog）として管理しており、この参照ログを見ることが出来る。

show

コミットの内容（日時、コミットメッセージ）をハッシュタグを指定してやる事で見れる。 また、ハッシュタグだけでなくタグ名やブランチ名を指定することでも見れる。

status

コミット状況を見る。
